线上音频转录（mac版）

我想解决的问题是：面对一个线上音频，暂时没法下载下来，但我想将这个音频转写成文字版。 一个验证下来比较便捷可行的思路是：先安装虚拟声卡，让系统声音通过虚拟声卡播放，用音频转录工具接收虚拟声卡的声音进行转录。 第一步：准备工具 虚拟声卡： BlackHole（建议下载 2ch 版本）。 转录工具： 通义听悟（支持浏览器实时录音转写） 或 飞书妙记。 第二步：核心配置（音频 MIDI 设置） 打开 Mac 自带的 “音频 MIDI 设置” 插件。 点击左下角 “+”，创建 “多输出设备”。 勾选 “MacBook Air 扬声器”（并设为主设备）和 “BlackHole 2ch”。 在 BlackHole 旁勾选 “漂移校正” 以保证音轨同步。 第三步：切换系统输出 点击 Mac 菜单栏的 系统设置/声音。 将输出设备切换为刚创建的 “多输出设备”。 注意：切换后键盘音量键将失效，需提前调好系统音量。 第四步：开启网页转录 权限检查： 确保浏览器已获得系统麦克风访问权限（路径：系统设置 -> 隐私与安全性 -> 麦克风）。 开始录音： 在通义听悟或飞书妙记中点击 “开始录音”。 设备选择： 在浏览器弹出的麦克风选项中，手动选择 “BlackHole 2ch (Virtual)”。 开始转录： 回到视频播放页面点击播放，转录页面将实时同步生成文字。 核查Tips 录制不到声音： 请检查视频播放器是否被静音，或浏览器标签页是否开启了静音模式。 录音中断： 长时间录制时，请确保关闭 Mac 的“自动睡眠”功能并接通电源。 权限失效： 若一切设置无误但仍无声音，可尝试重启浏览器以刷新隐私权限。

配置Github Pages

我想建立一个长期的写作基地，所以试着创建了自己的github pages，本篇记录了创建过程。 3分钟极简版，创建GitHub Pages 新建仓库： 登录 GitHub，点击右上角 + -> New repository。 关键步骤： 仓库名（Repository name）必须填写为 用户名.github.io。例如：用户名是 tom, 仓库名必须是 tom.github.io。 勾选 Public，点击 Create repository。 创建网页文件： 在仓库页面，点击 creating a new file 链接。 文件名填写：index.html 文件内容随便写，例如： HTML <h1>这是我的技术博客</h1> <p>Hello World! 2025年开始写作。</p> 滚动到底部，点击绿色按钮 Commit changes。 见证奇迹： 等待 1-2 分钟。 在浏览器输入地址：https://用户名.github.io。 应该能看到刚才写的网页了。 总结：GitHub Pages 的本质就是————你把文件放在仓库里，它自动帮你变成一个网站。
